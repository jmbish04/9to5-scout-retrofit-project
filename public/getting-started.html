Getting Started

Set up your 9to5-Scout AI-powered job discovery platform in minutes.

Prerequisites: You'll need a Cloudflare account with Workers, D1, R2, Vectorize, and Workers AI enabled.

1

Install Dependencies

First, make sure you have the required package manager and install project dependencies.

# Install pnpm globally if not available
npm install -g pnpm

# Navigate to project directory
cd 9to5-scout

# Install project dependencies
pnpm install

‚è±Ô∏è Note: Dependency installation may take up to 5 minutes. Don't cancel the process.

2

Configure Wrangler

Update wrangler.toml with your specific Cloudflare resource IDs:

# D1 Database
[[d1_databases]]
binding = "DB"
database_name = "JOB_SCRAPER_DB"
database_id = "your-d1-database-id"

# KV Namespace
[[kv_namespaces]]  
binding = "KV"
id = "your-kv-namespace-id"

# R2 Bucket
[[r2_buckets]]
binding = "R2"
bucket_name = "job-scraper"

# Vectorize Index
[[vectorize]]
binding = "VECTORIZE_INDEX" 
index_name = "jobs"

# Durable Objects
[[durable_objects.bindings]]
name = "SITE_CRAWLER"
class_name = "SiteCrawler"

[[durable_objects.bindings]]
name = "JOB_MONITOR" 
class_name = "JobMonitor"

# Workflows
[[workflows]]
binding = "DISCOVERY_WORKFLOW"
name = "DiscoveryWorkflow"

[[workflows]]
binding = "JOB_MONITOR_WORKFLOW"
name = "JobMonitorWorkflow"

[[workflows]]
binding = "CHANGE_ANALYSIS_WORKFLOW"
name = "ChangeAnalysisWorkflow"
3

Set Environment Variables

Configure these variables in your wrangler.toml [vars] section or as secrets:

# Authentication
API_AUTH_TOKEN = "your-secure-api-token"
BROWSER_RENDERING_TOKEN = "your-browser-rendering-token"

# Notifications
SLACK_WEBHOOK_URL = "your-slack-webhook-url"

# Email System (SMTP Configuration)
SMTP_ENDPOINT = "your-smtp-server-endpoint"
SMTP_USERNAME = "your-smtp-username" 
SMTP_PASSWORD = "your-smtp-password"

# Email Routing Configuration
EMAIL_ROUTING_DOMAIN = "9to5scout.dev"

üí° Security Tip: Use wrangler secret put for sensitive values like passwords and tokens.

4

Configure Email Routing

Set up Cloudflare Email Routing to receive job alerts from major platforms:

# Add to wrangler.toml email routing section
[[email]]
name = "job-alerts"
destination_addresses = ["*@9to5scout.dev"]

üìß Supported Email Addresses:

  • alerts@9to5scout.dev - LinkedIn and general job alerts
  • jobs@9to5scout.dev - Indeed and Monster job notifications
  • notifications@9to5scout.dev - Glassdoor and ZipRecruiter alerts
  • insights@9to5scout.dev - System-generated insights and reports
5

Run Database Migrations

Initialize the database schema with the required tables and indexes:

# Local development (for testing)
pnpm wrangler d1 migrations apply JOB_SCRAPER_DB --local

# Production deployment
pnpm wrangler d1 migrations apply JOB_SCRAPER_DB --remote
6

Deploy

Build and deploy your 9to5-Scout worker to Cloudflare:

# Build the project (includes type generation)
pnpm build

# Deploy to Cloudflare Workers
pnpm deploy
# or
pnpm wrangler deploy

üéâ Success! Your 9to5-Scout platform is now deployed and ready to use.

     }'
