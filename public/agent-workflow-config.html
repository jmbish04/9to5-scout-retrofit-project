Agent Configuration

Configure individual AI agents with custom parameters, prompts, and behavior settings.

üìã List Available Agents

# List all configured agents
curl -H "Authorization: Bearer <token>" \
     "https://your-worker.workers.dev/api/agents"

# Filter enabled agents only
curl -H "Authorization: Bearer <token>" \
     "https://your-worker.workers.dev/api/agents?enabled=true"

üõ†Ô∏è Create Custom Agent

curl -H "Authorization: Bearer <token>" \
     -H "Content-Type: application/json" \
     -X POST https://your-worker.workers.dev/api/agents \
     -d '{
       "id": "custom-job-analyzer",
       "name": "Custom Job Requirements Analyzer",
       "role": "Senior Job Analysis Expert",
       "goal": "Extract comprehensive job requirements and analyze candidate fit",
       "backstory": "Expert in parsing job descriptions with 15+ years of recruiting experience",
       "llm": "@cf/meta/llama-3.1-8b-instruct",
       "tools": ["web_search", "content_analysis"],
       "max_iter": 25,
       "max_rpm": 10,
       "verbose": false,
       "allow_delegation": false,
       "memory": true,
       "enabled": true
     }'

‚öôÔ∏è Configuration Parameters

Core Identity:

  • role - Agent's professional role
  • goal - Primary objective
  • backstory - Experience context

Behavior Settings:

  • max_iter - Maximum iterations (1-100)
  • verbose - Detailed output logging
  • memory - Context retention

AI Configuration:

  • llm - Language model selection
  • tools - Available tool integrations
  • max_rpm - Rate limiting

Templates:

  • system_template - System prompts
  • prompt_template - User prompts
  • response_template - Output format

Workflow Orchestration

Orchestrate multiple agents and tasks into comprehensive workflows for complex analysis.

üîÑ Sequential Workflows

Tasks execute one after another, with outputs passed between agents.

Example: Resume Analysis ‚Üí Job Matching ‚Üí Fit Scoring ‚Üí Report Generation

üèóÔ∏è Hierarchical Workflows

Manager agent coordinates multiple specialist agents working in parallel.

Example: Manager coordinates Resume, Job, and Company analysts simultaneously

‚öôÔ∏è Create Workflow

curl -H "Authorization: Bearer <token>" \
     -H "Content-Type: application/json" \
     -X POST https://your-worker.workers.dev/api/workflows \
     -d '{
       "id": "comprehensive-job-analysis",
       "name": "Comprehensive Job Analysis Workflow",
       "description": "Full analysis of job posting with resume fit assessment",
       "agents": [
         "resume-analyzer",
         "job-analyzer", 
         "company-researcher",
         "interview-strategist"
       ],
       "tasks": [
         "analyze-resume",
         "extract-job-requirements",
         "research-company",
         "generate-interview-guide"
       ],
       "process": "sequential",
       "verbose": false,
       "memory": true,
       "cache": true,
       "language": "en",
       "enabled": true
     }'

üöÄ Execute Workflow

curl -H "Authorization: Bearer <token>" \
     -H "Content-Type: application/json" \
     -X POST https://your-worker.workers.dev/api/workflows/comprehensive-job-analysis/execute \
     -d '{
       "context": {
         "job_url": "https://company.com/careers/senior-engineer",
         "applicant_id": "user-123",
         "resume_text": "Senior Software Engineer with 5+ years...",
         "target_role": "Senior Software Engineer"
       }
     }'
