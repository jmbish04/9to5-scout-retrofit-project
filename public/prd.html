Interview Prep: Mock Interview

Backend Prompt: Interview Prep

Goal: Build the backend functionality for an "Interview Prep" feature. This involves creating new database tables and API endpoints to manage interview stages, contacts, and AI-generated practice questions for a specific job application.

System: You are a senior backend engineer specializing in Cloudflare Workers, D1, and Hono. Your task is to implement the full backend for a new "Interview Prep" feature based on the following requirements. User: Please build out the backend for our Interview Prep feature. **1. Database Schema (D1 Migrations):** Create the following new tables: - `interview_stages`: - `id` (PK, text) - `job_id` (FK to `jobs.id`) - `user_id` (text) - `stage_name` (e.g., "Recruiter Screen", "On-site") - `date` (datetime) - `status` (e.g., "Scheduled", "Completed") - `interview_contacts`: - `id` (PK, text) - `job_id` (FK to `jobs.id`) - `name` (text) - `title` (text) - `email` (text) **2. API Endpoints (Hono Router):** Create the following REST endpoints, ensuring they are authenticated: - `GET /api/jobs/{jobId}/interviews`: List all interview stages and associated contacts for a job. - `POST /api/jobs/{jobId}/interviews`: Create a new interview stage. - `POST /api/jobs/{jobId}/interviews/contacts`: Add a new contact for the interview process. - `POST /api/jobs/{jobId}/interviews/generate-questions`: - This is an AI-powered endpoint. - It should take the `job_id` as input. - Internally, it will fetch the job description from the `jobs` table. - It will then call an LLM (e.g., via Workers AI) with a prompt like: "Based on this job description for a [Job Title] at [Company], generate 5 behavioral and 5 technical interview questions." - It should return the generated questions as a JSON object. **3. Response Schemas (Zod):** Define Zod schemas for the request bodies and response payloads for all new endpoints to ensure type safety.

Market Pulse Mockup

Backend Prompt: Market Pulse

Goal: Build backend functionality to aggregate and serve job market data. This involves creating new tables for aggregated statistics and endpoints to query them. This can be powered by a scheduled Worker that processes data from the `jobs` table.

System: You are a senior backend engineer specializing in Cloudflare Workers and D1. Your task is to implement the backend for a new "Market Pulse" feature. User: Please build out the backend for our Market Pulse feature. **1. Database Schema (D1 Migrations):** Create a new table for storing aggregated stats: - `market_stats`: - `id` (PK, text) - `date` (date, unique key with role & location) - `role` (text, e.g., "AI Engineer") - `location` (text, e.g., "San Francisco") - `avg_salary_min` (integer) - `avg_salary_max` (integer) - `top_skills` (JSON array of strings) - `job_count` (integer) **2. Scheduled Worker:** - Create a new Worker triggered by a cron schedule (e.g., daily). - This worker should: - Query the `jobs` table for recently active jobs. - Group jobs by role and location. - For each group, calculate the average min/max salary and count the occurrences of keywords/skills in the job descriptions. - `INSERT` or `UPDATE` the results into the `market_stats` table. **3. API Endpoints (Hono Router):** Create the following REST endpoints: - `GET /api/market/trends`: - Takes `role` and `location` as query parameters. - Returns a time-series of `market_stats` for that role/location combination (e.g., last 12 months). - `GET /api/market/skills`: - Takes `role` and `location` as query parameters. - Returns the latest `top_skills` data for that combination from `market_stats`.
